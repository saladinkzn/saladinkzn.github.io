Augmentor task

06 ноября 2014

Augmentor - это специальная таска для Gradle, которая нужна чтобы упростить жизнь разработчика. Augmentor позволяет оптимизировать рутинные операции, связанные с вызовом различных градловых тасок (gradle check, gradle installDebug, gradle build). Она позволяет вызывать задачу с помощью простого нажатия кнопки ENTER, а также умеет перезапускать задачу при изменениях в исходном коде.

Использование

Во-первых, необходимо подключить библиотеку с таской. Для этого необходимо добавить или отредактировать секцию buildscript, приведя её в соответствие с указанной ниже:

buildscript {
    repositories {
       maven { url 'http://oss.jfrog.org/artifactory/oss-snapshot-local' }
    }

    dependencies {
       classpath 'ru.shadam.augmentor:augmentor:0.1-SNAPSHOT'
    }
}

Затем, необходимо создать новый таск, используя подключенный AugmentorTask, например:

task re-build(type: ru.shadam.augmentor.AugmentorTask) {
    innerTask = 'build' // name of gradle augmentor task
    // Optional parameters:
    scanInterval = 10 // source changed throttling, -1 to disable, 0 for no throttling
    srcDirs = [ file('src/main/java') ] // list of directories to watch, project.sourceSets.main by default
}

Таким образом, мы получаем таск re-build, который можно запускать с помощью gradle re-build. Он будет следить за
изменениями в папке src/main/java и вызывать gradle build в случае обнаружения таковых.
